🤝 Sales Handover

  • Create project (Sales/Admin)
  • Save drafts, edit anytime
  • Fill: name, client, scope
  • Handover → triggers Onboarding
  • Auto-advance when fields complete

📋 Onboarding

  • Client data, requirements, theme
  • Auto-reminder toggle persists
  • Reminder cadence from Decision Policies
  • Max 10 reminders → HOLD state
  • Submit → ASSIGNMENT enqueued

📌 Assignment

  • PC creates & publishes tasks
  • Resource allocation per stage
  • AI-suggested task breakdown
  • Publish → BUILD enqueued
  • Multi-location support

🏗️ Build

  • Template clone from S3 / Git
  • Inject client data JSON
  • Build → requires dist/index.html
  • Upload preview to S3 (30d TTL)
  • HITL approval gate optional

🧪 Test

  • QA runner: Lighthouse + Axe
  • Tester uploads reports
  • Creates defect drafts
  • Pass → Complete or Defect Validation
  • WCAG 2 AA thresholds enforced

🐛 Defect Validation

  • AI agent analyses defects
  • Valid → back to Build (rework)
  • Invalid → back to Test (retest)
  • Cycle cap 5 → NEEDS_REVIEW
  • Admin/Manager resolves

⟳ Workflow Transition Paths

Sales→handover + fields complete→Onboarding
Onboarding→submit approved→Assignment
Assignment→published→Build
Build→success + HITL approved→Test
Test→defects found→Defect Validation
Test→no defects, pass→Complete
Defect Validation→valid defects→Build (rework)
Defect Validation→invalid / retest→Test
Defect Validation→all resolved→Complete ✓
Onboarding→10 reminders sent→HOLD 🔒
Defect Validation→cycle cap = 5→NEEDS_REVIEW ⚠️

🔒 HOLD State

Triggered after 10 onboarding reminders. Sweeper skips HOLD projects. Message: "Awaiting client response — attempted 10 times." Resume: client completes onboarding → Admin resumes autopilot.

⚠️ NEEDS_REVIEW State

Triggered when defect cycle cap (5) or build retry cap reached. Autopilot stops. Admin/Manager reviews, then re-enqueues or resumes autopilot to continue.

🧑‍💼 HITL Gate

Human-in-the-Loop gates configurable per stage. When enabled, pipeline pauses for Admin/Manager approval. Approve-build or Send-back endpoints used for manual review.

♻️ Recovery Actions

Enqueue (HITL projects) · Resume autopilot · Resolve confirmation portal · Advance via API. Sweeper (~90s) backstop: auto-enqueues ready stages, runs reminders.

⚡ Autonomous (zero-HITL) by default  ·  Sweeper runs every ~90s  ·  Complete stage → sends final email with CloudFront delivery URL + sentiment link  ·  Stage history logged with actor & timestamp
