DAISY — Job Execution Lifecycle

From project creation → Redis job queue → LangGraph worker → AI agents → S3 delivery  ·  Autonomous pipeline with sweeper backstop

⚡ Main Execution Flow
01 🤝 Project Created / Stage Ready Event trigger
Sales handover complete · Stage ready condition met · OR Sweeper detects ACTIVE project with no running job every ~90s. Auto-advance: previous stage success enqueues next.
02 📥 Job Enqueued to Redis Redis queue
POST /projects/{id}/advance · or pipeline auto-enqueues. Job record created in DB with status QUEUED. Stage job type: onboarding | assignment | build | test | defect_validation | complete.
03 🏭 Worker Polls Redis Queue Background worker
Docker worker service (delivery-worker) continuously polls Redis. Claims job → sets status RUNNING. Sends heartbeat. If heartbeat exceeds stage_timeout_minutes → marked FAILED by sweeper.
04 🧠 LangGraph Workflow Executes LangGraph 0.2.34
Worker invokes the LangGraph state machine for the current stage. Nodes: onboarding_node → assignment_node → build_hitl_node → test_node → defect_validation_node → complete_node. Each node delegates to a dedicated agent class.
05 🤖 AI Agent Runs Stage Logic GPT-4o
Agents in backend/app/agents/: OnboardingAgent · AssignmentAgent · BuildAgent · QAAgent · DefectAgent · CompletionAgent. Calls OpenAI GPT-4o (or FakeLLM fallback). Connector stubs: fetch_requirement(), fetch_external_task(), fetch_staging_url().
06 🏗️ Site Builder (Build stage) runners/site_builder
Download template zip from S3 (fallback: Git clone) · Unzip to temp dir · Inject client_contract.json · Build (requires dist/index.html) · Upload previews/{project_id}/{run_id}/site/ (30d TTL) · Copy to deliveries/{project_id}/current/site/
07 💾 Stage Output Persisted PostgreSQL
StageOutputs record written. Stage history entry created (from_stage, to_stage, at, actor_user_id). Artifacts uploaded. Job status → SUCCESS. Audit log entry created with request_id.
08 ⏩ Next Stage Enqueued Auto-advance
On success: transition to next stage, enqueue next job. If HITL gate enabled: pipeline pauses for human approval. If Complete: send final email (Resend) with CloudFront delivery URL + sentiment link.

⏱️ Sweeper Backstop (~90s interval)
Runs inside the worker process every ~90 seconds:
› Find ACTIVE projects with no running job
› Enqueue ready stages (autopilot projects only)
› Run onboarding reminders (cadence from Decision Policies)
› Mark projects HOLD after max reminders (default 10)
› Mark stuck RUNNING jobs as FAILED (timeout)
› Skips HOLD and NEEDS_REVIEW projects
🔒 Confirmation Portal Gate
Pipeline can block on ConfirmationRequest (fallback template / substitute artifact).
› Worker creates ConfirmationRequest → status: PENDING
› Client portal shows pending item (client_token auth)
› Daily reminder email (max 10 via Resend)
› Resolve: POST /confirmations/{id}/approve or /reject
› On decision: pipeline unblocks, next job enqueued

Download Template

S3: templates/{template_id}/{version}/template.zip (immutable) · Fallback: Git clone repo_url + repo_path if NoSuchKey

Unzip & Prepare

Single-root or nested ZIP · Inject JS helpers · Root-relative paths · Into temp working dir

Inject Client Data

Write data/client_contract.json · Client name, company, geo, requirements, theme, copy

Build (dist/)

Run build script (npm/build.sh) · MUST output dist/index.html · Fail-fast: list top-level dirs if missing

Upload Preview

S3: previews/{project_id}/{run_id}/site/ · Tagged expires=30d · CloudFront URL via get_preview_url()

Copy to Delivery

S3: deliveries/{project_id}/current/site/ (stable) · History: /run_id/site/ · Proof packs: artifacts/ (180d)
