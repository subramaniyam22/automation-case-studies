⟳ Stage-by-Stage Handoff Coordination
💼 Sales
Creates project with client details
Saves draft, edits fields
Triggers handover → auto-advance to Onboarding
Onboarding email sent to client (Resend)
🧑‍💼 Consultant
Receives project in Onboarding stage
Gathers client data, requirements, theme
Sends reminders (auto or manual)
Submits onboarding → Assignment enqueued
📋 PC
Receives project in Assignment
Creates tasks, allocates resources
Sets deadlines and owners
Publishes assignment → Build enqueued
🏗️ Builder
Worker pulls template from S3/Git
Builder updates build status
Uploads build artifacts
Completes build → Test enqueued (or HITL)
🧪 Tester
QA runner: Lighthouse + Axe auto
Tester uploads QA reports
Creates defect drafts if issues found
Pass → Complete or Defect Validation
🐛 Admin/AI
AI agent validates defects
Valid → routes back to Build
Invalid → routes back to Test
Cycle cap 5 → Admin reviews
✅ Complete
AI generates project summary
Delivery URL (CloudFront) generated
Final email sent to client (Resend)
Sentiment link + proof pack archived

🔐 RBAC Enforcement

  • JWT Bearer token on every request
  • Role decoded from JWT payload
  • Endpoint-level permission checks (rbac.py)
  • Consultant/PC: own projects only
  • Admin/Manager: all projects
  • Builder: build stage endpoints only
  • Tester: test stage endpoints only
  • Sales: sales stage + project creation
  • Preview auth: Bearer or ?access_token=
  • 6-hour token expiry (optional key rotation)

📊 Admin/Manager Control

  • System Config: SLA, thresholds, HITL
  • Decision Policies: reminder cadence, caps
  • Global Checklists: Build / QA / Defect
  • Preview Strategy: Static vs. Live
  • Operations Dashboard: job queue health
  • Quality Dashboard: sentiment tracking
  • Template Registry: publish/recommend
  • User Management: create/list users
  • Audit Logs: full action trail
  • Resume Autopilot / Enqueue (recovery)

🌐 Client Coordination

  • Sales enters client contact at creation
  • Onboarding email auto-sent on handover
  • Reminder emails: cadence from policies
  • Max 10 reminders → project HOLD
  • ConfirmationRequest blocks pipeline
  • Client portal: client_token (no login)
  • Approve / Reject with comment
  • Daily reminder email (max 10)
  • Completion: delivery URL + sentiment
  • Edit contacts: Admin/Consultant/PC
🔔 Notification, Webhook & Observability Flow

WebSocket Notifications

JWT-secured WS connection at /ws/notifications/{userId}. Real-time stage updates, job status, and HITL approval alerts to the frontend.

Email (Resend)

Onboarding email on Sales handover. Completion email with CloudFront URL. Client reminders (max 10, cadence configurable). Svix signature verification on webhook.

Chat Log Webhooks

AI Consultant chat logs delivered to external systems via CHAT_LOG_WEBHOOK_URL. X-Webhook-Secret header validated. Fallback to /api/webhooks/chat-logs.

Audit Logs

Every stage transition, reminder, HOLD/NEEDS_REVIEW, job enqueue/fail logged with actor, request_id, timestamp. Stage history: From, To, Date/Time, Triggered by (User/Pipeline).

Structured Logging

S3 upload keys, CloudFront URLs, and confirmation decisions all logged for observability. correlation_id per blueprint run. Worker heartbeat health: GET /system/health.
